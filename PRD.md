# 📄 PRD — Магазин японських ножів

**Версія:** 1.0  
**Статус:** В розробці (MVP)  
**Останнє оновлення:** Квітень 2025  
**Джерело правди:** Цей файл є єдиним джерелом правди для проекту. Будь-які зміни в логіці, структурі або вимогах фіксуються тут до початку написання коду.

---

## 1. Опис проекту

Інтернет-магазин японських кухонних ножів та аксесуарів для українського ринку. Спеціалізація — преміум-сегмент. Основна цільова аудиторія: кухарі-ентузіасти, шефи, колекціонери. Продаж відбувається двома шляхами: товари в наявності та ножі під замовлення (custom order).

**Мова інтерфейсу:** Українська  
**Валюта:** Гривня (UAH)  
**Доставка:** Нова Пошта (Україна)

## 3. Структура проекту (3 частини)

### 3.1 Public Part (Публічна зона)

Доступна без реєстрації. Основа для SEO та залучення клієнтів.

### 3.2 User Part (Клієнтська зона)

**Поточний стан:** Не реалізована. Авторизація та особистий кабінет відсутні. Замовлення оформлюється через форму без реєстрації.  
**Backlog:** Реалізувати в v2.

### 3.3 Admin Part (Бекофіс)

Payload CMS Admin Panel. Повне керування контентом і замовленнями.

## 4. Entities & Data Model (Схема БД)

### `products` — Товари

| Поле           | Тип                       | Опис                                                                                        |
| -------------- | ------------------------- | ------------------------------------------------------------------------------------------- |
| `title`        | text                      | Назва товару                                                                                |
| `slug`         | text (unique)             | URL-ідентифікатор, авто-генерується                                                         |
| `type`         | select                    | `knife` або `accessory`                                                                     |
| `status`       | select                    | `in_stock` (в наявності) або `custom_order` (під замовлення) — визначає розділ сайту та URL |
| `availability` | select                    | `available` або `unavailable` — чи можна замовити прямо зараз                               |
| `price`        | number                    | Ціна в грн (порожньо = ціна за запитом)                                                     |
| `category`     | relationship → categories | Категорія товару                                                                            |
| `images`       | upload[] → media          | Галерея зображень                                                                           |
| `description`  | richText                  | Детальний опис                                                                              |
| `specs`        | group (тільки для ножів)  | Характеристики: сталь, твердість, довжина клинка та руків'я, вага, обробка тощо             |

### `categories` — Категорії

Прості категорії для угруповання товарів у каталозі.

### `orders` — Замовлення

| Поле           | Тип           | Опис                                              |
| -------------- | ------------- | ------------------------------------------------- |
| `orderNumber`  | text          | Авто-генерований номер (#1001, #1002...)          |
| `name`         | text          | Ім'я покупця                                      |
| `phone`        | text          | Телефон (обов'язковий)                            |
| `email`        | text          | Email (необов'язковий)                            |
| `message`      | textarea      | Коментар до замовлення                            |
| `deliveryInfo` | textarea      | Дані для Нової Пошти                              |
| `items`        | array         | Список товарів: productId, title, price, quantity |
| `total`        | number        | Підсумкова сума в грн                             |
| `status`       | select        | `new` / `processing` / `completed` / `cancelled`  |
| `source`       | text          | Джерело: `contact_form` / `cart_checkout`         |
| `metadata`     | group         | IP, User-Agent, місто, країна (для безпеки)       |
| `honeypot`     | text (hidden) | Захист від ботів                                  |

### `posts` — Блог / Статті

| Поле            | Тип            | Опис              |
| --------------- | -------------- | ----------------- |
| `title`         | text           | Заголовок статті  |
| `slug`          | text           | URL-ідентифікатор |
| `coverImage`    | upload → media | Обкладинка        |
| `content`       | richText       | Тіло статті       |
| `publishedDate` | date           | Дата публікації   |

### `media` — Медіафайли

Централізоване сховище всіх зображень. ALT-текст синхронізується автоматично з назвою товару/статті (через хук `syncMediaAlt`).

### Globals

- **`SiteSettings`** — Глобальні налаштування сайту (контакти, соц.мережі, телефони).
- **`HomepageReviews`** — Відгуки клієнтів, що відображаються на головній сторінці.

## 5. Public Part — Вимоги

### Сторінки

| URL                       | Сторінка                  | Опис                                       |
| ------------------------- | ------------------------- | ------------------------------------------ |
| `/`                       | Головна                   | Hero-секція, переваги, категорії, відгуки  |
| `/knives/in_stock`        | Ножі в наявності          | Каталог ножів зі статусом `in_stock`       |
| `/knives/custom_order`    | Ножі під замовлення       | Каталог ножів зі статусом `custom_order`   |
| `/knives/[status]/[slug]` | Сторінка товару (ніж)     | Галерея, характеристики, кнопка замовлення |
| `/accessories`            | Аксесуари                 | Каталог аксесуарів                         |
| `/blog`                   | Блог                      | Список статей                              |
| `/blog/[slug]`            | Стаття                    | Повний текст статті                        |
| `/contacts`               | Контакти                  | Контактна форма + реквізити                |
| `/about`                  | Про нас                   | Опис магазину                              |
| `/shipping`               | Доставка і оплата         | Умови доставки                             |
| `/privacy`                | Політика конфіденційності | Правова сторінка                           |

### Каталог (ножі та аксесуари)

- Фільтрація за категорією
- Відображення ціни (або "Ціна за запитом")
- Відображення доступності (`available` / `unavailable`)
- При зміні `status` товару — автоматичний 301-редирект зі старого URL на новий

### Сторінка товару

- Фотогалерея з лайтбоксом (повноекранний перегляд)
- Таблиця характеристик (тільки для ножів)
- Кнопка "Замовити" — відкриває форму або переходить до чекауту
- Sticky (прилипаюча) панель з кнопкою на мобільних при скролі вниз
- SEO: динамічні `<title>`, `<meta description>`, JSON-LD `Product` schema, Open Graph

### Форма замовлення / Контакти

- Поля: Ім'я, Телефон (обов'язковий), Email, Повідомлення
- Honeypot захист від спамботів
- Після відправки: повідомлення "Дякуємо, ми зв'яжемося з вами" + запис в `orders`

## 6. Admin Part — Вимоги

Базується на стандартному Payload CMS Admin Panel.

### Доступи

- Тільки авторизовані адміністратори мають доступ до адмін-панелі
- Замовлення можуть читати/змінювати тільки адміни
- Товари і статті: доступні для читання публічно, керуються тільки адмінами

### Функціонал адміна

| Розділ       | Можливості                                                       |
| ------------ | ---------------------------------------------------------------- |
| Товари       | CRUD, масове завантаження фото (`BulkUploadDropzone`), авто-slug |
| Замовлення   | Список з фільтром за статусом, зміна статусу, перегляд деталей   |
| Блог         | CRUD статей з rich-text редактором                               |
| Медіа        | Централізована бібліотека файлів                                 |
| Налаштування | Редагування контактів, соцмереж (Global: SiteSettings)           |
| Відгуки      | Керування відгуками на головній (Global: HomepageReviews)        |

## 7. Background Processes & Integrations

| Процес           | Опис                                                                                  |
| ---------------- | ------------------------------------------------------------------------------------- |
| Auto-slug        | При створенні товару/статті slug генерується автоматично з назви                      |
| ALT-sync         | Після збереження товару ALT всіх зображень синхронізується з назвою товару            |
| ISR Revalidation | Після зміни товару/статті Next.js ревалідує відповідну сторінку (без повного ребілду) |
| Order numbering  | Номери замовлень генеруються автоматично (#1001, #1002...)                            |
| Spam protection  | Honeypot-поле приховане від користувача, видиме для ботів. Заповнення = блокування    |
| Analytics        | GA4 через GTM для трекінгу подій (перегляд товару, старт чекауту, замовлення)         |
| SEO              | Динамічні meta-теги, JSON-LD Product schema, sitemap.xml, robots.txt                  |
